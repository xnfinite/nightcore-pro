# AI Workflow Decision Map

Use this map before adding AI tools, automation platforms, agents, or compute infrastructure to a creator workflow.

## Quick Answer

Start with the workflow. If the steps are repeatable, a simple automation path may fit. If the workflow needs judgement, review, memory, or orchestration, define the human boundary before considering an agent. Use external compute only when the workload itself demands it.

## Path Map

| If the workflow needs... | Start with | Skip if |
| --- | --- | --- |
| Repeated app-to-app handoffs | [AI automation tool or custom agent](https://nightcore-pro.org/go/ai-automation-tool-or-custom-agent-launch) | The process changes every time or needs manual judgement first |

| A first decision across lanes | [Creator Path Finder](https://nightcore-pro.org/tools/creator-path-finder) | You already know the exact tool category |

## Practical Guardrails

- Automate stable steps, not fuzzy decisions.
- Keep a human review point wherever buyer trust or money is involved.
- Set a cost ceiling before adding external compute.
- Avoid adding agents before the workflow can be explained without AI.
